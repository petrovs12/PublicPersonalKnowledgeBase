Bloom Filters

Used to test for set membership. E.g. check if usernames have been used already. It's probabalistic, meaning that it'll return either definitely not in set or possibly in set. There's therefore a false positive rate ϵ\epsilonϵ involved.

Algorithm:

choose a number mmm, which is the number of bits in the filter. Choose kkk hash functions h1,h2,...,hk:S−>Zmh_1, h_2, ..., h_k: S->Z_mh1​,h2​,...,hk​:S−>Zm​, so they return numbers from 1tom1 to m1tom. Normally kkk is small, depending on ϵ\epsilonϵ and mmm depends on kkk and number of items to be added (how?idk).

  1. Initialize bit-array of size mmm with 0s.
  2. Operations: add(x),contains(x).
  3. add(x) calculates the kkk hashes, and gets some numbers h1(x),...h_1(x),...h1​(x),.... Set the bits h1(x),...h_1(x),...h1​(x),... to 1.
  4. query(x) calculates the kkk hashes, and gets some numbers h1(x),...h_1(x),...h1​(x),.... Check if the bits h1(x),...h_1(x),...h1​(x),... are 1. If all are, return True (possibly in set). Else, return False (definitely not in set).

Can choose mmm and kkk so that it's quite reliable, but still use it for non-critical applications. See (wikipedia)["https://en.wikipedia.org/wiki/Bloom_filter#Probability_of_false_positives"].

Use in Recommendation Systems

We can use bloom filters to check if we have already presented a given product to a given user. We can have a bloom filter for each user then and quickly filter if we should consider that for a given user.
